# 웹 전환 고민 정리

## 배경
- 외부접속/협업/배포편의를 위해 웹 전환 필요성 고려.
- 현재는 GUI(Electron/PyQt) 기반이며 로컬 엑셀을 자주 수정함.
- 웹 전환 시 파일 저장/동기화와 운영 비용에 대한 우려가 있음.

## 현재 파악된 사항
- 백엔드 언어는 Node 유지가 현실적(기존 로직 재사용).
- 서버 프레임워크는 Express가 무난.
- 파일 저장소는 Supabase Storage보다 R2가 용량/비용 측면에서 유리.
- R2는 객체 스토리지이므로 직접 수정은 불가:
  - 다운로드 → 로컬 수정 → 업로드(덮어쓰기) 흐름 필요.
- 엑셀 파일 크기는 최대 500KB 수준.
- 수정/저장은 하루 0~100회 수준.

## 웹 전환 리스크
- R2 업로드/다운로드 요청이 늘어나면 무료 한도 초과 가능.
- 웹 전환 시 운영 고려사항(스토리지/보안/배포/동기화)이 증가.
- Electron IPC 기반 로직을 HTTP API로 이관해야 함.

## 현실적인 대안(하이브리드)
- 기존 GUI는 유지.
- 협업/외부접속을 위한 웹 포털을 먼저 구축.
- GUI에서 저장 시 R2 자동 업로드로 동기화.
- 웹은 R2의 최신 파일을 읽어 조회/공유/처리.

## 남은 결정 사항
- 웹 전환 범위를 어디까지 가져갈지(전체/부분/하이브리드).
- 백엔드 배포처 선택(무료/유료, 슬립/콜드스타트 허용 여부).
- R2 사용 패턴(업로드 주기, 자동 업로드 방식).
