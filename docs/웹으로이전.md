# 웹 전환 인계 메모

## 배경
- 기존 프로젝트는 Electron 기반이며, 새 웹 버전은 **새 레포로 완전 분리**하기로 결정.
- 새 레포 이름: `accord-assistant-web`
- 기존 폴더를 복사 후 새 레포로 `git init` 및 원격 연결 완료, `main` 브랜치로 푸시 완료.

## 현재 결정 사항
- 웹 전환 목표: 기존 기능을 웹에서 재구현(코드는 별도 레포에서 작업).
- 배포 관련 논의 요약:
  - 소규모 사용(동시 4명, 소량 업/다운로드) 기준이면 무료 티어로 운영 가능.
  - 무료 백엔드는 슬립/콜드스타트, 로컬 파일 저장 휘발성 이슈가 있음.
  - Node 백엔드를 그대로 올리는 목적이면 Render가 현실적인 선택.
  - 안정성 우선 + 리팩터링 가능하면 Firebase가 더 안정적이지만 작업량 증가.
  - 파일 저장은 로컬 대신 외부 스토리지 고려 필요(예: R2/Supabase Storage 등).
- SMTP는 네이버 사용 중이며 20~30건 발송 규모는 문제 없다고 판단.

- 새 레포 init 및 push 완료

- 백엔드 배포처(Render/Firebase 중 선택) 및 파일 저장 전략 결정
